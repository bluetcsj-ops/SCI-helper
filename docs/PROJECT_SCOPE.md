# 项目范围说明

## 当前目标

建立一个完整但轻量的工程骨架，支持逐步开发以下核心能力：

- 五个 SCI 论文协作智能体：
  - Prof. RadOnc Mentor：研究趋势与课题建议
  - Dr. Vera Protocol：论文和实验方案制定
  - Dr. Data Lin：数据需求、统计分析与图表
  - Alex Writer：论文写作、文献与格式
  - Rev. Dr. Helena Skov：模拟审稿与返修应对
- Rhea Chen 作为页面级全流程实施监控员，负责项目执行、里程碑、延误风险、受阻任务和提醒策略，不作为普通咨询角色。
- 中央协调器 Orchestrator，负责智能体调用顺序和项目状态传递。
- 项目 A / Project B 并行管理。
- “我的研究书房”主控台。
- 后续接入真实服务：OpenAI API、PostgreSQL、Redis、Celery、Telegram、Email、DICOM/RT 数据处理、PubMed/Crossref 检索。

## 第一版原型边界

第一版不直接实现以下高风险或外部依赖能力：

- 不真实发送 Telegram 或邮件。
- 不连接真实 PostgreSQL。
- 不执行真实医学统计分析。
- 不解析真实 DICOM 文件。
- 不调用真实文献检索 API。
- 不自动生成投稿级 Word/PDF 文件。

这些能力会先以接口、服务类和配置项的形式预留。

## 成功标准

- 后端可以启动并返回健康检查。
- 前端可以启动并展示主控台。
- 页面可查看项目进度、下一里程碑和五个角色入口。
- 用户可以在原型聊天界面向任一角色发送消息，并收到模拟回复。
- 代码结构清楚，便于后续逐步替换为真实服务。
