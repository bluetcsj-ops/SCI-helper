# 开发计划

## 阶段 1：工程骨架

- 创建后端、前端、共享、基础设施和文档目录。
- 建立项目说明、架构草案和环境变量模板。
- 保留用户既往论文资料目录。

## 阶段 2：后端原型

- 创建 FastAPI 应用。
- 定义五个 Agent 的角色配置。
- 实现 Orchestrator 的基础状态流转。
- 提供项目、仪表盘和 Agent 对话 API。
- 使用内存数据模拟 Project A / Project B。

## 阶段 3：前端原型

- 创建 React + Vite 应用。
- 实现“我的研究书房”主控台。
- 展示 Project A / Project B 进度。
- 实现五个 Agent 快捷入口和模拟对话。

## 阶段 4：本地联调

- 前端调用后端 API。
- 验证健康检查、项目状态、角色对话和仪表盘数据。
- 补充运行说明。

## 阶段 5：真实服务接入

- 接入 OpenAI API。
- 接入 DeepSeek API，并保留 OpenAI 端可切换。
- 建立 Rhea 本地提醒与监控中心，后续可替换为 Celery + Telegram / Email 发送。
- 接入 PostgreSQL。
- 接入 Redis / Celery。
- 接入 Telegram / Email 通知。
- 接入文献检索与数据分析模块。

## 阶段 6：科研功能深化

- 研究趋势数据库。
- 数据需求规格生成。
- CSV 上传原型。
- 数据质量报告。
- 统计分析与图表输出。
- 论文写作模板。
- 模拟审稿与返修信件生成。

### 当前薄切片

- 已启动 Dr. Data Lin 数据工作区。
- 后端提供数据需求清单和 CSV 质量检查 API。
- 前端支持查看字段需求、上传脱敏 CSV、查看缺失字段、缺失率、唯一值、数值列范围和写作摘要。
- 前端支持选择分组列和结局列，生成描述性统计和分组比较草案。
- 前端支持基础图表预览，包括数值分布、分类构成和分组均值条形图。
- 前端支持将质控、统计和图表摘要交给 Alex Writer，生成 Results 段落草稿。
- 前端支持保存和恢复数据分析记录；后端保存报告 JSON，不保存原始 CSV。
- 前端支持将图表预览导出为 SVG 矢量文件。
- 统计报告内置可复现参数快照，并支持导出参数 JSON。
- 前端支持图表样式模板，并可导出 SVG、PNG 或通过打印视图另存 PDF。
- 统计报告增加变量分布检查、候选检验建议和 P 值计算边界提示。
- 质量报告增加脱敏与隐私检查，识别直接身份标识、编码标识符、日期标识符和高唯一文本列。
- 后端增加数据操作审计日志；上传质控、生成统计、隐私阻止和保存分析记录都会写入日志。
- 红色隐私风险会阻止统计草案生成；原型仍不保存原始 CSV。
- 统计报告生成标准库 Python 可复现实验脚本，可本地复现描述性统计、分组摘要和基础图表数据；脚本不自动计算 P 值。
- 后端增加演示用户、项目访问授权和 `viewer/editor/owner` 权限边界。
- 前端显示当前用户和当前项目权限；只读项目会禁用修改、上传和保存类操作。
- 增加正式假设检验执行前的人工确认流程；确认项缺失、隐私风险或复杂设计会阻止执行并写入审计日志。
- 当前正式检验原型支持两组数值结局的 Welch t 检验，结果可随分析记录保存并交给 Alex Writer。
- 已确认当前使用场景为单用户本地工具，因此不启用登录、SSO 和成员管理页面。

### 下一步

- 扩展更多正式检验类型，例如多组 ANOVA、非参数检验、配对设计和多重比较校正。
- 优化单用户体验：本地配置、项目归档、数据导入模板和一键导出。
