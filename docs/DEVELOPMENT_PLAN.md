# 开发计划

## 阶段 1：工程骨架

- 创建后端、前端、共享、基础设施和文档目录。
- 建立项目说明、架构草案和环境变量模板。
- 保留用户既往论文资料目录。

## 阶段 2：后端原型

- 创建 FastAPI 应用。
- 定义五个 Agent 的角色配置。
- 实现 Orchestrator 的基础状态流转。
- 提供项目、仪表盘和 Agent 对话 API。
- 使用内存数据模拟 Project A / Project B。

## 阶段 3：前端原型

- 创建 React + Vite 应用。
- 实现“我的研究书房”主控台。
- 展示 Project A / Project B 进度。
- 实现五个 Agent 快捷入口和模拟对话。

## 阶段 4：本地联调

- 前端调用后端 API。
- 验证健康检查、项目状态、角色对话和仪表盘数据。
- 补充运行说明。

## 阶段 5：真实服务接入

- 接入 OpenAI API。
- 接入 DeepSeek API，并保留 OpenAI 端可切换。
- 建立 Rhea 本地提醒与监控中心，后续可替换为 Celery + Telegram / Email 发送。
- 接入 PostgreSQL。
- 接入 Redis / Celery。
- 接入 Telegram / Email 通知。
- 接入文献检索与数据分析模块。

## 阶段 6：科研功能深化

- 研究趋势数据库。
- 数据需求规格生成。
- 数据质量报告。
- 统计分析与图表输出。
- 论文写作模板。
- 模拟审稿与返修信件生成。
